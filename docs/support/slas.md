# Support SLAs

This document defines support expectations for Metamorphic Guard.

## Channels
- GitHub Issues for bug reports and feature requests
- Discussions for Q&A and design proposals

## Response Targets (business days)
- Critical (build broken, data loss): initial response within 1 day
- High (blocking production usage): initial response within 2 days
- Medium (non-blocking bug): initial response within 4 days
- Low (feature request, docs): initial response within 5 days

## Resolution Targets (best-effort)
- Critical: fix or workaround within 3 days
- High: fix or workaround within 7 days
- Medium: scheduled in next minor release
- Low: triage and roadmap tagging

## Scope
- Supported Python versions per `compatibility.md`
- Official features documented in user guides and API reference
- Third-party plugins: best-effort guidance

## Security
- Security issues receive priority handling; report privately if possible

## Maintenance Windows
- Releases typically on weekdays; emergency patches as needed

## Version Support
- We support the latest MAJOR.MINOR and the previous MINOR for critical fixes
